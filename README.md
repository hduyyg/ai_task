# 简介

这是一个AI任务管理服务，用户在网页上维护需求列表，AI获取到任务之后自动执行。

更简单的说：一个AI agent套壳项目，提前做好各种配置，让用户能更便捷的与Agent进行交互。

# 理念

## 1.只关注AI执行任务的结果，不关注过程

​	用户在网页端新建/编辑任务，AI客户端从网页后端获取任务并在处理之后把生成产物返回给网页后端，在网页前端展示出来。

​	整个过程中，不会有AI客户端执行过程展示，有的只是：任务描述、任务是否完成、任务完成后的产物。

​	AI客户端返回的产物，是用户对于任务需要关注的结果。以软件开发为例，产物主要是：开发文档（方便直接看文档知道AI做了什么）、AI执行回复（AI本轮工作总结）、代码merge列表（实际修改的代码列表）

## 2. 极致个性化的 ToC

​	我一直想要做的，是极致个性化的 ToC 产品，是每个人都可以有一个自己喜爱的版本。

​	在AI开发时代，每个人不需要有代码知识也能做开发，让这一切成为可能。

​	所以，我更希望开源的，是我的工作方式、我的工作理念，至于项目代码、产品设计，其实不重要。

​	这个项目，凡有你不喜欢的、不符合你的习惯的，尽管用AI修改一个专属你自己的版本。我不认为一个产品设计要非常通用，千人千面正是我的愿景。我所提供的就是一个初始模板，让你在自己DIY时减少成本而已。

## 3. 细分场景优化，不做通用

​	在细分场景下做优化才能得到最极致的体验，和AI说最少得话就能把事情给办了。

​	不限定细分场景的情况下，要让 AI 能够准确识别意图，一定要填写补充知识的，比如让AI整理篮球明星，他可能告诉你是蔡徐坤。

​	在高频工作里，每个任务都给 AI 补充这种背景知识，重复写就太没必要了。

​	所以通过客户端的方式，每个客户端代表一种细分场景，在创建任务的时候选择客户端，就不必再告诉AI当前任务是什么场景、背景知识有哪些。
